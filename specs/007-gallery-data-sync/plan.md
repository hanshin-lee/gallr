# Implementation Plan: Gallery Data Sync from Google Sheets

**Branch**: `007-gallery-data-sync` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-gallery-data-sync/spec.md`

---

## Summary

Connect the gallr KMP app to a live Supabase `exhibitions` table by configuring the existing `ExhibitionApiClient` with Supabase PostgREST credentials and swapping the active `StubExhibitionRepository` for `ExhibitionRepositoryImpl`. A Google Apps Script sync pipeline reads a curator-managed Google Sheet and writes validated rows into Supabase on every edit and on a 5-minute backup trigger.

Zero new KMP dependencies required — the project already has Ktor 2.3.12, `ExhibitionApiClient`, `ExhibitionRepositoryImpl`, and `ExhibitionDto` fully wired.

---

## Technical Context

**Language/Version**: Kotlin 2.1.20 (KMP shared + platform modules); Google Apps Script V8 runtime (sync pipeline)
**Primary Dependencies**:
- Ktor 2.3.12 (`ktor-client-core`, `ktor-client-content-negotiation`, `ktor-serialization-kotlinx-json`, `ktor-client-okhttp`, `ktor-client-darwin`) — already present, no changes
- Supabase PostgREST REST API — no SDK, called via plain HTTP with two auth headers
- `kotlinx-datetime` — already present (date parsing in domain model)
- Google Apps Script `UrlFetchApp` — built into GAS runtime, no import needed

**Storage**: Supabase Postgres (hosted); `exhibitions` table (see `data-model.md` for schema)
**Testing**: Kotlin test (unit tests for `ExhibitionApiClient` and `ExhibitionDto` mapping); GAS sync script tested manually via test sheet row
**Target Platform**: Android (minSdk 24+), iOS 16+ (via KMP); Google Apps Script (server-side V8)
**Project Type**: mobile-app (KMP) + server-side sync script (GAS)
**Performance Goals**: ≤500 rows synced in under 6 minutes (GAS execution limit); app list and map views remain responsive at 500 rows
**Constraints**:
- Ktor 2.3.12 — must NOT be upgraded (supabase-kt 3.x requires Ktor 3.x — rejected per research Decision 1)
- Supabase anon key — safe to bundle in app binary (RLS ensures SELECT-only access)
- Service role key — MUST stay in GAS Script Properties only, never in app binary or git
- GAS time-based trigger fires at most every 5 minutes (platform limit)

**Scale/Scope**: ≤500 exhibition rows; 2 platforms (Android + iOS); 1 sync script
