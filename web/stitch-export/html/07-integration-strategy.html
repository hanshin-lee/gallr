# Technical Integration Plan: gallr Website (Eleventy + Supabase) This document outlines the steps to translate the Google Stitch designs into the production Eleventy environment, specifically for **Approach C: Stitch + Live Content Preview**. ## 1. Data Architecture (Supabase to Eleventy) To enable the "This Week" live preview, we need to bridge the Supabase `exhibitions` table with Eleventy's static build process. ### Supabase Table Schema (Required Fields) - `id`: UUID - `title`: String (Exhibition name) - `gallery_name`: String (Linked from `galleries` table) - `start_date`: Date - `end_date`: Date - `city`: String (e.g., "Seoul") - `district`: String (e.g., "Hannam-dong") - `editorial_summary`: Text (The "one-liner" for the website) - `is_featured`: Boolean (To flag items for the homepage hero/edit) - `image_url`: URL (Optional: if we decide to show exhibition images) ### Eleventy Data Fetch (`web/_data/exhibitions.js`) ```javascript const { createClient } = require('@supabase/supabase-js'); const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_KEY); module.exports = async function() { const { data, error } = await supabase .from('exhibitions') .select('*, galleries(name, district)') .eq('city', 'Seoul') .gte('end_date', new Date().toISOString()) .order('start_date', { ascending: false }) .limit(4); if (error) { console.error('Supabase fetch error:', error); return []; // Fallback to empty or cached data } return data; }; ``` ## 2.
